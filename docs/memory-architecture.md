# 记忆系统架构

## 设计灵感

本系统受人类记忆机制启发：

1. **海马体 → 每日日记**：短期记忆暂存（memory/YYYY-MM-DD.md）
2. **大脑皮层 → MEMORY.md**：长期记忆存储
3. **睡眠巩固 → daily-meditation**：提取重要内容到长期记忆
4. **遗忘曲线 → 衰减公式**：琐碎记忆自然衰退

## 记忆分层

```
┌─────────────────────────────────────┐
│      MEMORY.md（长期记忆）           │
│  ┌─────────────────────────────┐   │
│  │ 📌 静态配置区（永不衰减）    │   │
│  │ - 铁律 (99 分)           │   │
│  │ - 敏感凭证 (15 分)           │   │
│  │ - 系统配置 (12 分)           │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 📅 动态记忆区（参与衰减）    │   │
│  │ - 重要事件 (7-13 分)         │   │
│  │ - 周洞察 (9-10 分)           │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
           ↑ 提炼（≥7 分）
┌─────────────────────────────────────┐
│   memory/YYYY-MM-DD.md（短期记忆）  │
│   - 每日日记（30 天后清理）         │
│   - 条目级打分（### 标题 <!-- importance: X -->）│
└─────────────────────────────────────┘
```

## 打分机制

### 评分标准

| 层级 | 分数 | 内容类型 | 衰减 |
|------|------|---------|------|
| 核心记忆 | 99 | 铁律 | ❌ 永不衰减 |
| 核心记忆 | 12-15 | 账号、密码、API Key | ❌ 永不衰减 |
| 重要记忆 | 7-10 | 配置、决策、安全规则 | ✅ 正常衰减 |
| 普通记忆 | 4-6 | 日常日志、一般事项 | ✅ 正常衰减 |
| 琐碎记忆 | 1-3 | 闲聊、测试 | ✅ 加速衰减 |

### 打分流程

```
memory-scoring.py（每天 02:00）
    ↓
扫描 memory/*.md 文件
    ↓
匹配 ### 或 #### 标题
    ↓
提取条目内容
    ↓
关键词匹配 → 返回权重
    ↓
插入 <!-- importance: X -->
```

## 衰减机制

### 衰减公式

```python
effective_score = importance × e^(-0.01×days_old)
```

**参数**：
- `λ = 0.01`（衰减系数）
- 30 天后分数降至 74%
- 90 天后分数降至 37%

### 衰减曲线

```
分数
  |
15|  ━━━━━━━━━━━━━━━━━━━  核心记忆（永不衰减）
  |

  |   ╲

  |     ╲

  |       ╲

  0       30      90
```

## 清理机制

### 清理条件

| 位置 | 条件 | 操作 |
|------|------|------|
| MEMORY.md | effective_score < 2 | 删除条目 |
| memory/*.md | days_old > 30 | trash 删除文件 |

### 清理流程

```
memory-decay-check.py（每天 02:00）
    ↓
解析 MEMORY.md 动态记忆区
    ↓
计算每条 effective_score
    ↓
标记 effective_score < 2 的条目
    ↓
生成清理清单 → 飞书群
    ↓
等待主人确认
    ↓
执行清理（trash 命令）
```

### 保护机制

1. **提炼优先**：daily-meditation 先提取重要内容（≥7 分）到 MEMORY.md
2. **可恢复**：使用 `trash` 而非 `rm`（废纸篓可恢复）
3. **主人确认**：清理前必须生成清单并等待确认
4. **核心保护**：importance ≥ 11 永不衰减

## 日常流程

```
02:00 daily-memory-maintenance
    ↓
1. memory-scoring.py
   - 为新条目打分
   - 格式：### 标题 <!-- importance: X -->
    ↓
2. memory-decay-check.py
   - 检查 MEMORY.md 动态记忆区
   - 计算 effective_score
   - 标记可清理条目
    ↓
3. 生成清理清单
   - MEMORY.md 低分条目
   - memory/30 天 + 文件
    ↓
4. 飞书群 → 等待确认

04:00 daily-meditation
    ↓
1. 读取昨天的 memory/YYYY-MM-DD.md
    ↓
2. 提取 importance ≥ 7 的条目
    ↓
3. 复制到 MEMORY.md 动态记忆区
    ↓
4. 发布 WordPress 日记
```
