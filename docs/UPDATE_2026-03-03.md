# 📝 更新说明 (2026-03-03)

## 修复内容

### memory-scoring.py - 严重 Bug 修复 ✅

**问题**：
- 原代码使用 `content.replace(title, new_title, 1)` 仅替换第一次出现的标题
- 当文件中有多个相同日期的标题时（如多个 `#### 2026-03-01`），会导致：
  - 第一个标题被反复插入评分标签
  - 文件内容错位、截断、覆盖
  - **导致 MEMORY.md 文件损坏，Gateway 持续重启**

**修复方案**：
- 改用**倒序遍历** + **精确索引替换**
- 使用 `re.finditer()` 获取每个标题的精确位置
- 通过字符串切片精确替换，不影响其他内容

**修复后的核心代码**：
```python
# 倒序遍历，避免字符串长度改变影响后续 match 的 index
for match in reversed(matches):
    title = match.group(1).strip()
    
    # 检查是否已有评分
    if '<!-- importance:' in title:
        continue
    
    # 计算评分
    importance = calculate_importance(entry_content)
    new_title = f"{title} <!-- importance: {importance} -->"
    
    # 精确替换该 match 位置的字符串
    start_idx = int(match.start(1))
    end_idx = int(match.end(1))
    content = content[:start_idx] + new_title + content[end_idx:]
```
